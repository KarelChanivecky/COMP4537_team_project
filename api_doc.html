Gurden Angi A01073916 Karel Chanivecky A01052674 Description of the Resources and URIs


1. POST /user/create

- It’s a POST method that allows users to sign up into the application, with email, password passed as body in the request. It sends jwt token as response.


2. POST /user/login

- It’s a POST method that allows the user to log into the application. If the user enters incorrect credentials or doesn’t exist, alerts/ warnings are thrown. It sends jwt token as response.


3. GET /endpoints
(auth required)

- It’s a GET method that fetches the count of total number of hits for all API routes.


4. GET /lists
(auth required)

- It’s a GET method that all fetches all the TO-DO lists of the user. Response is a JSON object


5. GET /lists/{ listId }/items
(auth required) - It’s a GET method that all fetches all the list items of a particular list of the user. Response is a JSON object


6. POST /lists
(auth required)

- It’s a POST method to create new TO-DO List. Accepts list description (string) in the request body


7. POST /lists/{ listId }/items
(auth required)

- It’s a POST method to create new TO-DO List Item in a particular list. Accepts item description (string) in the request body.


8. PUT /lists/{ listId }
(auth required)

- It’s a PUT method to update existing TO-DO List. Accepts list description (string) in the request body


9. PUT /lists/{ listId }/items/{ itemId }
(auth required)

- It’s a PUT method to update existing TO-DO List Item in a particular list. Accepts item description (string) in the request body.


10. DELETE /lists/{ listId }
(auth required)

- It’s a DELETE method to delete existing TO-DO List of a user.


11. DELETE /lists/{ listId }/items/{ itemId }
(auth required)

- It’s a DELETE method to delete existing TO-DO List Item in a particular list.
